A Signup Form containing a label and input for:

  • The user’s first name
  • The user’s last name
  • The user’s email address
  • The user’s date of birth
  • The user’s password
  • A submit button

Requirements:

  • Create each form in it’s own HTML document
  • Use an external stylesheet. Do not use inline CSS
  • Create a single stylesheet or one for each form
  • Style each form to look dazzling using CSS. Lack of customization may be interpreted as low effort.
  • Feel free to improve each form by adding additional elements. For example, adding a paragraph to explain a Credit/Debit Card’s CSV would be a User Experience improvement.

Note: The pattern attribute of the input tag is not supported in Internet Explorer 9 and earlier versions, or in Opera Mini.

Regarding styling to look "dazzling", I chose styling to look crisp, clean, and easy to interpret - hopefully that serves as appropriate "dazzle."

I chose to style based roughly on Google's Material Design.

Additionally, I think for a form that is to be a web component, it should be somewhat generically styled since forms are about gathering information, and therefore should be more about clarity and usability.

I have put two different Birth Date input fields on this form - the first is a text field with a pattern for client-side validation, and the second is a date field - since the date field is not fully supported nor does it render reliably with applied styling. Firefox, and browsers under iOS do not seem to honor styling for type=date.

I chose to consider and use client-side validation for format, where appropriate, at the advice from developer.mozilla.org, where the first paragraph reads:

While you should always validate data on your servers, but additional validation of data on the Web page itself has multiple benefits. In many ways, users are annoyed by forms. By validating form data while the user is filling it out, the user can know immediately if they've made any mistakes; this saves the time of waiting for an HTTP response, and saves your server from dealing with bad form input.
